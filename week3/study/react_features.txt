1. DOM (기존 방식)
Document Object Model의 약어 
객체를 문서 구조로 표현하는 방법으로 XML이나 HTML로 작성
웹 브라우저는 DOM을 활용하여 객체에 자바스크립트와 CSS 적용
DOM 자체를 읽고 쓸때는 빠르나 웹 브라이주 단에서 DOM의 변화가 일어나면 웹 브라우저가 CSS를 다시 연산, 레이아웃 구성, 페이지 리페인트에 시간이 소요
정적이므로 페이지를 새로 그림

2. Virtual DOM (리액트)
DOM을 추상화한 자바스크립트 객체
DOM의 사본

DOM 업데이트 과정
1. 데이터 업데이트 시 전체 UI를 Virtual DOM에 리렌더링
2. 이전 Virtual DOM에 있던 내용과 현재 내용을 비교
3. 바뀐 부분만 DOM에 적용

3. 리액트는 라이브러리
다른 웹 프레임워크가 Ajax, 데이터 모델링, 라우팅 등과 같은 기능을 내장하고 있는 반면 리액트는 뷰만 신경쓰는 라이브러리
AngularJS, Backbone.js 등의 프레임워크와 함께 사용 가능
